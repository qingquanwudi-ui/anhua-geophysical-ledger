# 物探检测台账系统

这是一个用于管理湖南安化抽水蓄能电站施工期物探检测台账资料的本地 Web 系统。系统支持台账文件导入、在线预览、查询、统计、异常检查和统计结果导出，适合工程资料整理、周报/月报/季报统计和后续多人协作维护。

## 主要功能

- 台账文件导入与版本保存。
- 台账信息在线预览和单元格查看。
- 高级查询：按标段、检测类型、委托单位、文件名、日期范围、报告编号、委托编号、单位工程、工程部位、检测结果筛选。
- 异常数据检查：检查报告编号重复、委托编号重复、日期缺失、检测结果缺失、数量异常、文件名与标段或委托类型不一致等问题。
- 检测数据统计：支持周报、月报、季报、年报和自定义时间范围。
- 季报/年报综合统计：选择季报或年报后，自动追加合同量完成情况、单位工程检测覆盖情况和各检测项目正文统计表，统计表可在页面查看并导出 Word、Excel。
- 全检测类型统计：覆盖锚杆无损检测、锚杆拉拔试验、钻孔摄像检测、锚索多循环张拉、预应力锚杆张拉、洞室松弛圈检测、桩身完整性检测、回填灌浆质量单孔注浆试验、弹性波检测、钻孔成像。
- 统计导出：支持 Word 和 Excel 导出。大范围或全量统计建议使用 Excel。
- 用户登录与账号管理。

## 启动方式

在 Windows 下运行：

```bat
work\start_ledger_server.bat
```

默认访问：

```text
http://127.0.0.1:8765/
```

如果默认端口被占用，程序会尝试备用端口：

```text
http://127.0.0.1:8766/
```

## 数据目录

系统运行数据默认读取和保存到：

```text
outputs/ledger_system/ledger_system.db
outputs/ledger_system/storage/
```

其中：

- `ledger_system.db` 是系统数据库。
- `storage/` 保存上传后的台账原始文件和版本文件。

## Git 管理范围

Git 只管理代码和说明文档。以下运行数据不纳入 Git，需要单独备份或交付：

```text
outputs/ledger_system/ledger_system.db
outputs/ledger_system/storage/
```

如果把 GitHub 代码发给其他人，对方能拿到系统代码，但不会自动拿到你的本地台账数据库和上传文件。需要一起使用同一批台账数据时，应额外交付上述 `outputs/ledger_system/` 数据目录，或让对方重新导入台账文件。

## 协作维护建议

多人维护时建议通过 GitHub 协作：

1. 每个人先拉取最新代码。
2. 修改代码后本地运行验证。
3. 提交 Git commit。
4. 推送到 GitHub。
5. 其他人再拉取最新代码。

代码更新后，本地网站内容不会自动变化，必须重新启动 `work\start_ledger_server.bat`，浏览器刷新后才能看到新功能。
