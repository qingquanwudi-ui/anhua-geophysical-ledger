# 物探检测台账系统需求说明

## 1. 项目背景

本项目用于管理“湖南安化抽水蓄能电站施工期物探检测项目部”的检测台账资料。

系统当前围绕 Excel 台账文件、SQLite 数据库、Web 管理页面和统计导出功能展开，目标是把分散的检测台账资料统一归档、查询、统计和导出。

## 2. 当前项目范围

当前系统主要处理物探检测相关台账，包含但不限于以下检测类型：

- 锚杆无损检测
- 锚杆拉拔试验
- 钻孔摄像检测
- 锚索多循环张拉试验
- 基桩完整性检测
- 回填灌浆质量单孔注浆试验检测

系统已支持多个项目或标段资料管理，例如 Q2、C1、C2、C3、C4 等。

## 3. 已完成功能

### 3.1 Excel 台账解析

系统可以读取物探检测台账 Excel 文件，自动识别工作表、表头、检测类型、检测日期、报告编号、委托编号、工程部位、检测结果、施工数量、检测数量等字段。

对于不同检测项目中不统一的字段，系统会作为扩展字段保存，避免原始信息丢失。

### 3.2 SQLite 数据库存储

系统已能将台账数据写入 SQLite 数据库。

当前已有两类数据库：

- 离线台账数据库：用于脚本导入、查询和 Excel 导出。
- Web 系统数据库：用于 Web 上传文件、版本管理、模板保存、用户管理和统计。

核心数据包括：

- 台账文件信息
- 文件版本信息
- Excel 工作簿、工作表、单元格、合并区域、行高列宽
- 用户和登录会话

### 3.3 Excel 统计导出

系统已能根据台账数据生成 Excel 统计结果，包含：

- 按检测项目统计
- 按检测日期排序
- 查询结果导出
- 原始台账模板恢复

### 3.4 Excel 模板保存与恢复

系统可以把 Excel 的单元格内容、公式、样式、合并区域、行高、列宽等信息保存到数据库，并支持从数据库恢复为 Excel 文件。

### 3.5 Web 台账管理系统

当前 Web 系统已实现：

- 登录
- 用户管理
- 文件上传
- 文件版本管理
- 原始文件下载
- 台账预览
- 工作表查看
- 管理员在线编辑部分表格数据
- 检测数据统计
- 单位工程统计
- Word 统计报告导出

Web 应用入口文件为：

```text
work/ledger_web_app.py
```

启动脚本为：

```text
work/start_ledger_server.bat
```

### 3.6 部署路径修复

启动脚本已修复旧电脑绝对路径问题。

当前脚本会根据自身所在位置自动定位项目根目录、Web 应用文件和日志目录。项目文件夹整体移动到其他位置后，不需要再修改项目路径。

## 4. 当前运行方式

在 Windows 下，进入项目文件夹后运行：

```bat
work\start_ledger_server.bat
```

默认访问地址：

```text
http://127.0.0.1:8765
```

备用端口：

```text
http://127.0.0.1:8766
```

健康检查地址：

```text
http://127.0.0.1:8765/health
```

## 5. 交付给其他用户时的注意事项

如果把项目交给其他用户使用，应完整发送整个项目文件夹，不能只发送 `work` 目录。

必须包含：

```text
work/
outputs/
outputs/ledger_system/ledger_system.db
outputs/ledger_system/storage/
```

对方电脑需要满足以下条件：

- Windows 系统
- 可用的 Python 3
- Python 中已安装 `openpyxl`
- Python 中已安装 `python-docx`

启动脚本会自动尝试寻找：

- Codex Desktop 自带 Python
- 系统 PATH 中的 `python.exe`
- Windows Python Launcher `py -3`

如果对方电脑没有 Python 或缺少依赖，需要先安装运行环境。
