# 物探检测台账管理系统需求说明

## 1. 项目背景

本项目用于管理湖南安化抽水蓄能电站施工期物探检测项目部相关台账资料。系统目标是把分散的 Excel 台账、上传文件、检测记录和统计结果统一管理，减少人工翻表、复制和汇总工作，提高工程资料查询、统计、质检和交接维护效率。

系统当前形态为本地 Web 应用，用户通过浏览器访问本机服务。代码通过 GitHub 管理，实际台账数据库和上传文件保存在本地 `outputs/ledger_system/` 目录中。

## 2. 当前项目范围

系统主要处理物探检测台账，当前已覆盖以下检测类型：

- 锚杆无损检测
- 锚杆拉拔试验
- 钻孔摄像检测
- 锚索多循环张拉
- 预应力锚杆张拉
- 洞室松弛圈检测
- 桩身完整性检测
- 回填灌浆质量单孔注浆试验
- 弹性波检测
- 钻孔成像

系统支持 Q2、C1、C2、C3、C4 等标段资料管理，并可按施工、监理、业主等委托类别进行统计。

## 3. 已完成功能

### 3.1 台账文件管理

- 支持上传 Excel 台账文件。
- 保存文件名称、项目名称、标段、委托单位、专业类型等元信息。
- 支持文件版本管理，同一个台账再次上传会形成新版本。
- 支持原始文件下载。
- 管理员可删除文件。

### 3.2 Excel 解析与原表查看

- 自动解析 Excel 工作簿、工作表、单元格、公式、合并单元格、行高、列宽等信息。
- 支持在网页中查看原表内容。
- 支持按文件和工作表切换查看。
- 保留原始台账信息，避免导入后关键字段丢失。

### 3.3 高级查询

支持按以下条件筛选台账记录：

- 标段
- 委托单位
- 检测类型
- 文件名
- 日期范围
- 报告编号
- 委托编号
- 单位工程
- 工程部位
- 检测结果

查询结果可导出为 Excel，便于进一步整理和移交。

### 3.4 数据质检

当前质检重点检查明确影响使用的数据问题：

- 报告编号缺失
- 委托编号缺失
- 检测日期缺失或无法识别
- 检测结果缺失
- 单位工程和工程部位均为空
- 施工数量、检测数量异常
- 文件名与标段不一致
- 文件名与委托类型不一致
- 文件没有解析出工作表

说明：报告编号或委托编号重复不再视为异常。因为同一编号可能同时出现在施工台账、监理台账或不同负责单位资料中，也可能对应多行工程部位或不同负责数量。

### 3.5 检测数据统计

检测数据统计用于周报、月报和常规检测数量统计，主要按检测项目、标段、委托类别和时间范围汇总。

支持内容：

- 周报、月报、季报、年报和自定义时间范围筛选。
- 按检测类型生成检测数量、施工数量、检测比例、检测成果等统计表。
- 支持 Word 导出。
- 支持 Excel 导出。
- 支持选择需要导出的统计表。

### 3.6 单位工程统计

单位工程统计用于季度、年度等更偏工程报表口径的综合统计。

支持内容：

- 按单位工程筛选。
- 生成合同量完成情况统计。
- 生成单位工程检测覆盖情况统计。
- 生成季度、年度综合统计表。
- 支持 Word 导出。
- 支持 Excel 导出。

### 3.7 统计字段优化

已优化锚杆无损检测中的杆长和直径识别：

- 杆长可从“锚杆长度”“锚索长度”“杆长”等字段识别。
- 直径可从“锚杆直径”“锚索直径”或“规格/型号”中提取。
- 支持 `Ф25/Ф28`、`HRB400E/28/25`、`4.5m/6m/12m` 等常见格式。
- 避免把 `30根`、`φ28` 等内容误识别为杆长。

### 3.8 用户与权限

- 支持登录。
- 支持管理员和普通用户。
- 管理员可创建用户、重置密码、删除文件。
- 默认管理员账号见 README。

### 3.9 部署路径修复

启动脚本已经去除旧电脑绝对路径，改为根据脚本所在位置自动识别项目根目录。项目文件夹整体移动到其他电脑或其他目录后，一般不需要修改路径即可启动。

## 4. 当前运行方式

在项目根目录运行：

```bat
work\start_ledger_server.bat
```

默认访问地址：

```text
http://127.0.0.1:8765/
```

备用访问地址：

```text
http://127.0.0.1:8766/
```

健康检查地址：

```text
http://127.0.0.1:8765/health
```

## 5. 数据与交付说明

GitHub 仓库主要管理代码和说明文件。以下运行数据默认不提交到 Git：

```text
outputs/ledger_system/ledger_system.db
outputs/ledger_system/storage/
```

如果只把 GitHub 代码发给别人，对方可以运行系统，但看不到你的本地台账数据。若希望对方看到同一批台账数据，需要同时交付整个 `outputs/ledger_system/` 目录，或至少交付：

```text
outputs/ledger_system/ledger_system.db
outputs/ledger_system/storage/
```

对方需要放到项目根目录下对应位置。

## 6. 当前仍可优化的问题

### 6.1 数据备份与恢复

建议增加一键备份和恢复功能，备份对象包括：

- `outputs/ledger_system/ledger_system.db`
- `outputs/ledger_system/storage/`

可进一步支持每日自动备份、备份列表、恢复前确认等功能。

### 6.2 操作日志

建议增加操作日志，记录：

- 用户登录
- 上传文件
- 删除文件
- 下载文件
- 修改数据
- 重置密码
- 导出报表

日志应包含操作人、操作时间、操作对象和操作结果，方便追溯。

### 6.3 权限细化

当前已有管理员和普通用户。后续可细化为：

- 管理员：用户管理、文件删除、数据修改、系统维护。
- 上传人员：上传和更新台账。
- 查看人员：查询、预览、下载和导出。
- 审核人员：查看质检结果并确认数据。

### 6.4 报表模板化

当前已经能生成 Word 和 Excel 统计表。后续可进一步做成更正式的固定模板：

- 周报模板
- 月报模板
- 季报模板
- 年报模板
- 分标段统计模板
- 分检测类型统计模板

模板化后可以减少人工调整 Word 排版的工作。

### 6.5 数据导入校验

当前质检在导入后检查。后续可增加上传时预校验，例如：

- 文件名与标段是否匹配。
- 委托单位类型是否与文件名匹配。
- 关键字段是否存在。
- 日期格式是否可识别。
- 数量字段是否能转为数字。

### 6.6 易用性优化

后续可以继续优化页面交互：

- 统计页增加更清晰的功能分区。
- 导出入口进一步固定在页面上方。
- 长表格增加折叠、搜索和定位。
- 数据质检结果增加“忽略/确认”状态。
- 上传后给出更明确的解析结果提示。

### 6.7 打包为免安装版本

如果要交给不熟悉 Python 的普通用户使用，建议后续打包为免安装版本：

- 自带 Python 环境。
- 自带依赖包。
- 双击即可启动。
- 启动后自动打开浏览器。

## 7. 维护建议

多人协作建议使用 GitHub：

1. 修改前先拉取最新代码。
2. 修改后本地启动系统验证。
3. 提交 Git commit。
4. 推送到 GitHub。
5. 其他维护人员拉取后重启本地服务。

需要注意：代码更新不会自动同步运行数据；运行数据需要单独备份、复制或重新导入。

## 8. 当前优先级建议

建议后续按以下顺序推进：

1. 数据备份与恢复。
2. 操作日志。
3. 质检结果确认和忽略机制。
4. 报表模板化。
5. 权限细化。
6. 上传前数据校验。
7. 免安装打包。

当前系统已经具备基础使用闭环，可以完成台账上传、查看、查询、质检、统计和导出。下一阶段重点应放在数据安全、长期维护、权限控制和报表规范化上。
